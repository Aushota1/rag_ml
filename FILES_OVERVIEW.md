# Обзор файлов проекта

## Основные компоненты (Python)

### Конфигурация
- **config.py** (874 bytes)
  - Центральная конфигурация системы
  - Параметры моделей, путей, поиска
  - Настройки через переменные окружения

### Парсинг и обработка документов
- **parser.py** (4,243 bytes)
  - Извлечение текста из PDF
  - OCR для сканированных документов
  - Извлечение метаданных (законы, дела, даты)

- **chunker.py** (4,803 bytes)
  - Структурный чанкинг документов
  - Поиск границ (Article, Section)
  - Сохранение иерархии и контекста

### Индексация и поиск
- **indexer.py** (6,456 bytes)
  - Гибридная индексация (FAISS + BM25)
  - Векторные эмбеддинги
  - Сохранение/загрузка индексов

- **retriever.py** (3,783 bytes)
  - Гибридный поиск
  - Комбинирование оценок
  - Дедупликация результатов

- **reranker.py** (1,572 bytes)
  - Реранкинг с cross-encoder
  - Точная оценка релевантности

### Обработка запросов
- **query_rewriter.py** (3,761 bytes)
  - Переформулировка запросов
  - Расширение синонимами
  - Генерация альтернативных формулировок

### Генерация ответов
- **generator.py** (7,616 bytes)
  - Базовая генерация ответов
  - Эвристическая экстракция
  - Поддержка всех типов ответов

- **llm_integration.py** (8,914 bytes)
  - Интеграция с OpenAI API
  - Улучшенная генерация с LLM
  - Fallback на эвристики

### Основной пайплайн
- **pipeline.py** (4,281 bytes)
  - Оркестрация всех компонентов
  - Обработка вопросов end-to-end
  - Сбор телеметрии

### API сервер
- **api.py** (3,509 bytes)
  - FastAPI REST API
  - Эндпоинты: /answer, /batch, /health
  - Обработка запросов

## Зависимости между файлами

```
config.py
    ↓
parser.py → chunker.py → indexer.py
                              ↓
query_rewriter.py → retriever.py ← reranker.py
                        ↓
                   generator.py ← llm_integration.py
                        ↓
                   pipeline.py
                        ↓
                     api.py

build_index.py использует:
    - parser.py
    - chunker.py
    - indexer.py

test_pipeline.py использует:
    - pipeline.py
    - public_dataset.json

run.sh/bat запускает:
    - build_index.py (если нужно)
    - api.py
```

## Как начать работу

1. **Изучите документацию:**
   - Начните с `README.md`
   - Затем `QUICKSTART.md` для быстрого старта
   - `ARCHITECTURE.md` для понимания системы

2. **Установите и запустите:**
   - Используйте `run.sh` (Linux/Mac) или `run.bat` (Windows)
   - Или следуйте инструкциям в `QUICKSTART.md`

3. **Тестируйте:**
   - Запустите `test_pipeline.py`
   - Попробуйте примеры из `EXAMPLES.md`

4. **Настройте:**
   - Отредактируйте `config.py`
   - Создайте `.env` из `.env.example`

5. **Разверните:**
   - Следуйте `DEPLOYMENT.md` для продакшн развертывания
