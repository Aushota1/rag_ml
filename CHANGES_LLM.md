# Изменения: Интеграция LLM

## Что изменилось

### 1. Конфигурация (`config.py`)
Добавлены новые параметры:
- `USE_LLM` - включить/выключить LLM (по умолчанию: true)
- `LLM_PROVIDER` - провайдер LLM (по умолчанию: openai)
- `LLM_MODEL` - модель для использования (по умолчанию: gpt-4o-mini)

### 2. Pipeline (`pipeline.py`)
- Автоматический выбор генератора (LLM или эвристики)
- Импорт `EnhancedAnswerGenerator` из `llm_integration.py`
- Добавление `model_name` в телеметрию
- Логирование используемой модели при инициализации

### 3. Генерация submission (`hack/generate_submission.py`)
- Загрузка вопросов из `questions.json` вместо `public_dataset.json`
- Использование `model_name` из телеметрии
- Динамическое описание архитектуры с указанием модели

### 4. Зависимости (`requirements.txt`)
- Добавлен `python-dotenv>=1.0.0` для загрузки .env файлов

### 5. Конфигурация окружения (`.env.example`)
Обновлен с новыми параметрами:
```env
USE_LLM=true
LLM_PROVIDER=openai
LLM_MODEL=gpt-4o-mini
OPENAI_API_KEY=your_openai_api_key_here
```

### 6. Новые файлы
- `LLM_SETUP.md` - детальная инструкция по настройке
- `QUICK_LLM_GUIDE.md` - быстрый старт
- `test_llm.py` - скрипт проверки конфигурации
- `CHANGES_LLM.md` - этот файл

## Как использовать

### С LLM (рекомендуется):
1. Создайте `.env` файл с вашим OpenAI API ключом
2. Запустите: `python hack/generate_submission.py`
3. Система использует gpt-4o-mini для генерации ответов

### Без LLM:
1. В `.env` установите `USE_LLM=false`
2. Система использует эвристическую генерацию

## Преимущества

✅ Более точные ответы на сложные вопросы
✅ Лучшее понимание юридического контекста
✅ Корректная обработка всех типов ответов
✅ Автоматическое форматирование в нужный формат
✅ Graceful fallback на эвристики при ошибках

## Обратная совместимость

Все изменения обратно совместимы:
- Если `USE_LLM=false` или нет API ключа → используются эвристики
- Старые скрипты продолжают работать
- Формат submission.json не изменился
