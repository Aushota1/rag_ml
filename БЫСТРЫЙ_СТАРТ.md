# 🚀 Быстрый старт RAG системы

## 📋 Два режима работы

RAG система поддерживает два режима:

1. **Standard Mode** - быстрый, простой (рекомендуется для начала)
2. **Tokenier Mode** - с классификацией, более точный

## ⚡ Вариант 1: Standard Mode (5 минут)

### Шаг 1: Установка зависимостей

```bash
pip install -r requirements.txt
```

### Шаг 2: Настройка

```bash
# Скопируйте .env.example в .env
cp .env.example .env

# Отредактируйте .env (укажите путь к документам и API ключ)
# DOCUMENTS_PATH=путь/к/документам
# OPENAI_API_KEY=ваш_ключ
```

### Шаг 3: Построение индекса

```bash
python build_index.py --mode standard
```

### Шаг 4: Запуск

```bash
# Вариант 1: API сервер
python api.py

# Вариант 2: Быстрый тест
python quick_test.py
```

**Готово!** Система работает в стандартном режиме.

---

## 🎯 Вариант 2: Tokenier Mode (30 минут)

### Шаг 1: Проверка файлов tokenier

```bash
# Проверьте наличие файлов
python check_tokenier_setup.py
```

Должны быть файлы:
- `models/tokenier/tokenizer.pkl`
- `models/tokenier/embedding.pth`

Если файлов нет, см. `SETUP_TOKENIER_FILES.md`

### Шаг 2: Обучение классификаторов

```bash
# Обучить все модели (10-20 минут)
python train_tokenier_models.py --model all

# Или по отдельности:
python train_tokenier_models.py --model document
python train_tokenier_models.py --model question
python train_tokenier_models.py --model relevance
```

### Шаг 3: Построение индекса с tokenier

```bash
python build_index.py --mode tokenier
```

### Шаг 4: Запуск

```bash
# API сервер с tokenier
python api.py
```

**Готово!** Система работает с улучшенным качеством.

## 📊 Что выбрать?

### Используйте Standard Mode если:
- ✅ Первый запуск системы
- ✅ Нужно быстро протестировать
- ✅ Нет времени на обучение моделей
- ✅ Базового качества достаточно

### Используйте Tokenier Mode если:
- ✅ Production окружение
- ✅ Нужно максимальное качество
- ✅ Есть размеченные данные
- ✅ Готовы потратить время на обучение

## 🛠️ Полезные команды

### Проверка настройки

```bash
# Проверить файлы tokenier
python check_tokenier_setup.py

# Проверить индекс
python quick_test.py
```

### Построение индекса

```bash
# Standard mode
python build_index.py --mode standard

# Tokenier mode
python build_index.py --mode tokenier

# С переменной окружения
export USE_TOKENIER=true
python build_index.py
```

### Обучение моделей

```bash
# Все модели
python train_tokenier_models.py --model all

# Только document classifier
python train_tokenier_models.py --model document

# Только question classifier
python train_tokenier_models.py --model question

# Только relevance classifier
python train_tokenier_models.py --model relevance
```

### Сравнение режимов

```bash
# Сравнить standard vs tokenier
python compare_indexing_modes.py

# Результаты сохраняются в comparison_results.json
```

## 🐛 Troubleshooting

### Ошибка: "Documents path not found"

```bash
# Укажите правильный путь в .env
DOCUMENTS_PATH=C:/Users/Aushota/Downloads/dataset_documents
```

### Ошибка: "Tokenizer not found"

```bash
# Проверьте файлы
python check_tokenier_setup.py

# Скопируйте файлы если нужно
# См. SETUP_TOKENIER_FILES.md
```

### Ошибка: "No PDF files found"

```bash
# Убедитесь, что PDF файлы в правильной папке
ls $DOCUMENTS_PATH/*.pdf
```

### Медленное построение индекса

```bash
# Используйте standard mode для быстрого тестирования
python build_index.py --mode standard

# Или уменьшите количество документов
```
