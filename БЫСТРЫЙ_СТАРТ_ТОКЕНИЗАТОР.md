# 🚀 Быстрый старт: Обучение токенизатора

## Шаг 1: Проверка PDF парсинга

Проверьте что PDF файлы читаются корректно:

```bash
python test_pdf_parsing.py --directory "C:/Users/Aushota/Downloads/dataset_documents" --max-files 5
```

Вы увидите:
- ✓ Сколько файлов успешно распарсилось
- ✗ Какие файлы не удалось прочитать
- Статистику по количеству символов

## Шаг 2: Извлечение корпуса

Извлеките текст из всех PDF в один файл:

```bash
python extract_pdf_corpus.py
```

Это создаст:
- `data/corpus.txt` - текстовый файл с документами
- `data/corpus.json` - метаданные

## Шаг 3: Обучение токенизатора

Обучите BPE токенизатор на корпусе:

```bash
python train_bpe_tokenizer.py --corpus data/corpus.txt --vocab-size 30000 --test
```

Параметры:
- `--vocab-size 30000` - размер словаря (можно 10000-50000)
- `--test` - протестировать после обучения

Результат:
- `models/tokenier/tokenizer.pkl` - обученная модель

## Альтернатива: Всё в одной команде

Если не хотите создавать промежуточный корпус:

```bash
python train_bpe_tokenizer.py --documents-path "C:/Users/Aushota/Downloads/dataset_documents" --vocab-size 30000 --test
```

⚠️ Это медленнее, т.к. PDF парсятся каждый раз заново.

## Что дальше?

После обучения токенизатора:

1. Используйте его в RAG pipeline
2. Интегрируйте с другими компонентами
3. Обучите embedding layer (если нужно)

## Troubleshooting

### Ошибка: No PDF files found
```bash
# Проверьте путь
ls "C:/Users/Aushota/Downloads/dataset_documents"
```

### Ошибка: No text extracted
- PDF может быть защищен
- Включите OCR в `parser.py`: `self.ocr_enabled = True`

### Слишком долго
- Уменьшите `--vocab-size` до 10000
- Используйте меньше документов

### Прервать обучение
- Нажмите `Ctrl+C` - прогресс сохранится автоматически
