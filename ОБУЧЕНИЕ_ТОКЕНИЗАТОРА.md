# Обучение токенизатора на PDF документах

## Быстрый старт

### Вариант 1: Напрямую из PDF (медленнее, но проще)

```bash
python train_bpe_tokenizer.py --documents-path "C:/Users/Aushota/Downloads/dataset_documents" --vocab-size 30000 --test
```

### Вариант 2: Через корпус (быстрее, рекомендуется)

1. Извлечь текст из PDF в корпус:
```bash
python extract_pdf_corpus.py --documents-path "C:/Users/Aushota/Downloads/dataset_documents" --output data/corpus.txt
```

2. Обучить токенизатор на корпусе:
```bash
python train_bpe_tokenizer.py --corpus data/corpus.txt --vocab-size 30000 --test
```

## Параметры

### extract_pdf_corpus.py

- `--documents-path` - путь к директории с PDF файлами
- `--output` - путь для сохранения корпуса (по умолчанию: `data/corpus.txt`)
- `--min-length` - минимальная длина текста документа в символах (по умолчанию: 100)

### train_bpe_tokenizer.py

- `--corpus` - путь к файлу корпуса (взаимоисключающий с --documents-path)
- `--documents-path` - путь к директории с PDF (взаимоисключающий с --corpus)
- `--vocab-size` - размер словаря (по умолчанию: 30000)
- `--output` - путь для сохранения модели (по умолчанию: `models/tokenier/tokenizer.pkl`)
- `--checkpoint-interval` - интервал сохранения чекпоинтов (по умолчанию: 100)
- `--test` - протестировать токенизатор после обучения

## Что происходит

### Шаг 1: Извлечение текста (extract_pdf_corpus.py)

1. Находит все PDF файлы в указанной директории
2. Парсит каждый PDF с помощью `DocumentParser`
3. Извлекает текст со всех страниц
4. Фильтрует документы по минимальной длине
5. Сохраняет корпус в текстовый файл (один документ на строку)
6. Создает файл метаданных с статистикой

### Шаг 2: Обучение токенизатора (train_bpe_tokenizer.py)

1. Загружает корпус или парсит PDF напрямую
2. Инициализирует BPE токенизатор
3. Обучает на текстах (BPE алгоритм)
4. Сохраняет чекпоинты каждые N итераций
5. Сохраняет финальную модель
6. (Опционально) Тестирует на примерах

## Результаты

После обучения вы получите:

- `models/tokenier/tokenizer.pkl` - обученный токенизатор
- `data/corpus.txt` - извлеченный корпус (если использовали extract_pdf_corpus.py)
- `data/corpus.json` - метаданные корпуса

## Статистика

Скрипты выводят полезную статистику:

- Количество обработанных документов
- Количество неудачных парсингов
- Общее количество символов
- Средняя длина документа
- Размер словаря
- Compression ratio (символов на токен)

## Примеры использования

### Минимальный пример
```bash
python extract_pdf_corpus.py
python train_bpe_tokenizer.py --corpus data/corpus.txt
```

### С кастомными параметрами
```bash
python extract_pdf_corpus.py \
  --documents-path "D:/documents" \
  --output "my_corpus.txt" \
  --min-length 200

python train_bpe_tokenizer.py \
  --corpus "my_corpus.txt" \
  --vocab-size 50000 \
  --output "models/my_tokenizer.pkl" \
  --checkpoint-interval 50 \
  --test
```

### Прямо из PDF (без промежуточного корпуса)
```bash
python train_bpe_tokenizer.py \
  --documents-path "C:/Users/Aushota/Downloads/dataset_documents" \
  --vocab-size 30000 \
  --test
```

## Troubleshooting

### Ошибка: No PDF files found
- Проверьте путь к директории
- Убедитесь что в директории есть .pdf файлы

### Ошибка: No text extracted
- PDF может быть защищен или поврежден
- Попробуйте включить OCR в `parser.py` (установите `self.ocr_enabled = True`)

### Слишком долго обучается
- Уменьшите `--vocab-size`
- Используйте меньше документов
- Увеличьте `--checkpoint-interval`

### Прерывание обучения
- Нажмите Ctrl+C - прогресс сохранится
- Модель будет сохранена в последнем чекпоинте
