# 📊 Визуальная схема процесса индексации

## Полный поток данных

```
┌─────────────────────────────────────────────────────────────────┐
│                    ВХОДНЫЕ ДАННЫЕ                                │
│  📁 Папка с PDF документами (~200 файлов, ~50-500 страниц)     │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│              ЭТАП 1: ПАРСИНГ (DocumentParser)                    │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  Для каждого PDF:                                        │   │
│  │  1. Открыть файл (pypdf)                                 │   │
│  │  2. Извлечь текст со всех страниц                        │   │
│  │  3. OCR для сканов (опционально)                         │   │
│  │  4. Извлечь метаданные (regex)                           │   │
│  │     - Название, номер закона/дела, дата                  │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                   │
│  Выход: ~200 документов с текстом и метаданными                 │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│          ЭТАП 2: ЧАНКИНГ (StructuralChunker)                     │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  Для каждого документа:                                  │   │
│  │  1. Найти структурные элементы:                          │   │
│  │     • Article X                                          │   │
│  │     • Section Y                                          │   │
│  │     • Нумерованные пункты                                │   │
│  │  2. Разбить по границам структуры                        │   │
│  │  3. Если чанк > 512 токенов:                             │   │
│  │     → Разбить с overlap 50 токенов                       │   │
│  │  4. Добавить метаданные к каждому чанку                  │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                   │
│  Выход: ~15000 чанков с метаданными                             │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│    ЭТАП 3: ВЕКТОРИЗАЦИЯ (HybridIndexer - Vector)                │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  1. Загрузить модель:                                    │   │
│  │     SentenceTransformer('all-MiniLM-L6-v2')              │   │
│  │                                                           │   │
│  │  2. Создать эмбеддинги (батчами):                        │   │
│  │     embeddings = model.encode(texts, batch_size=32)      │   │
│  │     Результат: [15000, 384] float32                      │   │
│  │                                                           │   │
│  │  3. Нормализовать векторы:                               │   │
│  │     faiss.normalize_L2(embeddings)                       │   │
│  │     Каждый вектор теперь длины 1                         │   │
│  │                                                           │   │
│  │  4. Создать FAISS индекс:                                │   │
│  │     index = IndexFlatIP(384)                             │   │
│  │     index.add(embeddings)                                │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                   │
│  Выход: vector.index (~23 MB)                                   │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│      ЭТАП 4: BM25 ИНДЕКС (HybridIndexer - Lexical)              │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  1. Токенизировать тексты:                               │   │
│  │     tokens = [text.lower().split() for text in texts]   │   │
│  │                                                           │   │
│  │  2. Создать BM25 индекс:                                 │   │
│  │     bm25 = BM25Okapi(tokens)                             │   │
│  │     Вычисляет TF-IDF статистику                          │   │
│  │                                                           │   │
│  │  3. Сохранить словарь и статистику                       │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                   │
│  Выход: bm25.pkl (~10 MB)                                       │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│           ЭТАП 5: СОХРАНЕНИЕ НА ДИСК                             │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  Сохранить 4 файла:                                      │   │
│  │  1. vector.index  - FAISS индекс                         │   │
│  │  2. bm25.pkl      - BM25 индекс                          │   │
│  │  3. chunks.pkl    - Чанки с метаданными                  │   │
│  │  4. texts.pkl     - Тексты чанков                        │   │
│  └──────────────────────────────────────────────────────────┘   │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│                    ИТОГОВЫЙ РЕЗУЛЬТАТ                            │
│  📦 index/ папка (~56 MB)                                       │
│     ├── vector.index  (23 MB) - Векторный поиск                │
│     ├── bm25.pkl      (10 MB) - Лексический поиск              │
│     ├── chunks.pkl    (15 MB) - Метаданные                     │
│     └── texts.pkl     (8 MB)  - Тексты                         │
│                                                                   │
│  ✅ Готово к использованию в RAG pipeline                       │
└─────────────────────────────────────────────────────────────────┘
```

## Использование индексов при поиске

```
┌─────────────────────────────────────────────────────────────────┐
│                    ЗАПРОС ПОЛЬЗОВАТЕЛЯ                           │
│  "What is Article 1 of the Operating Law?"                      │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ├──────────────────┬──────────────────────┐
                         ▼                  ▼                      ▼
              ┌──────────────────┐ ┌──────────────────┐ ┌──────────────────┐
              │  ВЕКТОРНЫЙ ПОИСК │ │   BM25 ПОИСК     │ │  МЕТАДАННЫЕ      │
              │                  │ │                  │ │                  │
              │ 1. Эмбеддинг     │ │ 1. Токенизация   │ │ Для каждого      │
              │    запроса       │ │    запроса       │ │ найденного чанка │
              │                  │ │                  │ │ извлекаем:       │
              │ 2. Поиск в FAISS │ │ 2. BM25 скоринг  │ │ - doc_id         │
              │    top-20        │ │    top-20        │ │ - page           │
              │                  │ │                  │ │ - hierarchy      │
              │ 3. Косинусное    │ │ 3. TF-IDF оценки │ │ - title          │
              │    сходство      │ │                  │ │                  │
              └────────┬─────────┘ └────────┬─────────┘ └────────┬─────────┘
                       │                    │                    │
                       └────────────────────┼────────────────────┘
                                            ▼
                              ┌──────────────────────────┐
                              │  ГИБРИДНОЕ ОБЪЕДИНЕНИЕ   │
                              │                          │
                              │ combined_score =         │
                              │   α × vector_score +     │
                              │   (1-α) × bm25_score     │
                              │                          │
                              │ α = 0.5 (по умолчанию)   │
                              └────────────┬─────────────┘
                                           ▼
                              ┌──────────────────────────┐
                              │  TOP-40 КАНДИДАТОВ       │
                              │  (для реранкинга)        │
                              └──────────────────────────┘
```

## Детальная схема создания эмбеддинга

```
Текст чанка: "Article 1. Definitions In this Law..."
     │
     ▼
┌─────────────────────────────────────────────────────────┐
│  ТОКЕНИЗАЦИЯ (WordPiece)                                │
│  ["[CLS]", "Article", "1", ".", "Def", "##initions",   │
│   "In", "this", "Law", "...", "[SEP]"]                 │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  BERT ENCODER (6 слоев)                                 │
│  Каждый токен → 384-мерный вектор                       │
│  [CLS]: [0.12, -0.34, ...]                             │
│  "Article": [0.45, 0.23, ...]                          │
│  ...                                                    │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  MEAN POOLING                                           │
│  Усредняем векторы всех токенов                         │
│  embedding = mean([vec_cls, vec_article, ...])          │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  L2 НОРМАЛИЗАЦИЯ                                        │
│  embedding = embedding / ||embedding||                  │
│  Теперь длина вектора = 1                               │
└────────────────────────┬────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  ИТОГОВЫЙ ЭМБЕДДИНГ                                     │
│  [0.023, -0.145, 0.089, ..., 0.234]  (384 числа)       │
│  Сохраняется в FAISS индекс                             │
└─────────────────────────────────────────────────────────┘
```

## Сравнение методов поиска

```
┌──────────────────┬──────────────────┬──────────────────┐
│  ВЕКТОРНЫЙ ПОИСК │    BM25 ПОИСК    │ ГИБРИДНЫЙ ПОИСК  │
├──────────────────┼──────────────────┼──────────────────┤
│ Семантическое    │ Точное           │ Лучшее из обоих  │
│ понимание        │ совпадение слов  │ миров            │
│                  │                  │                  │
│ "закон о труде"  │ "employment law" │ Находит оба      │
│ → находит        │ → находит только │ варианта         │
│ "employment law" │ точное совпадение│                  │

│ ✅ Синонимы      │ ❌ Синонимы      │ ✅ Синонимы      │
│ ✅ Перефразы     │ ❌ Перефразы     │ ✅ Перефразы     │
│ ❌ Редкие термины│ ✅ Редкие термины│ ✅ Редкие термины│
```
