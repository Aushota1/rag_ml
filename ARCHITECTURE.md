# Архитектура RAG системы

## Обзор

Система построена по двухэтапной архитектуре:
1. **Индексация** - предварительная обработка документов (выполняется один раз)
2. **Инференс** - обработка запросов в реальном времени

## Детальный поток данных

### Этап 1: Индексация

```
PDF документы
    ↓
[DocumentParser]
    ├─ Извлечение текста (pypdf)
    ├─ OCR для сканов (pytesseract)
    └─ Разбивка по страницам
    ↓
[MetadataExtractor]
    ├─ Название документа
    ├─ Дата принятия
    ├─ Номер закона/дела
    └─ Юрисдикция
    ↓
[StructuralChunker]
    ├─ Поиск структурных элементов (Article, Section)
    ├─ Разбивка по границам
    ├─ Сохранение иерархии
    └─ Overlap для контекста
    ↓
[HybridIndexer]
    ├─ Векторный индекс (FAISS)
    │   ├─ Embeddings (sentence-transformers)
    │   ├─ Нормализация L2
    │   └─ IndexFlatIP (косинусное сходство)
    └─ BM25 индекс
        ├─ Токенизация
        └─ TF-IDF статистика
    ↓
Сохранение на диск
```

### Этап 2: Инференс

```
Вопрос пользователя
    ↓
[QueryRewriter]
    ├─ Расширение синонимами
    ├─ Упрощение до ключевых слов
    └─ Переформулировка (2-3 варианта)
    ↓
[HybridRetriever]
    ├─ Векторный поиск (top-20)
    ├─ BM25 поиск (top-20)
    ├─ Комбинирование оценок (α=0.5)
    ├─ Дедупликация
    └─ Top-40 кандидатов
    ↓
[Reranker]
    ├─ Cross-encoder оценка
    ├─ Попарное сравнение (query, document)
    └─ Top-5 лучших
    ↓
[ThresholdValidator]
    ├─ Проверка max_score >= 0.3
    └─ has_info: true/false
    ↓
[AnswerGenerator]
    ├─ Сборка контекста из чанков
    ├─ Формирование промпта
    ├─ Вызов LLM (опционально)
    └─ Форматирование ответа
    ↓
[TelemetryCollector]
    ├─ TTFT (time to first token)
    ├─ Total time
    ├─ Token usage
    └─ Retrieved pages
    ↓
JSON ответ
```

## Компоненты системы

### 1. DocumentParser (`parser.py`)

**Назначение:** Извлечение текста из PDF документов

**Технологии:**
- `pypdf` - извлечение текста из PDF
- `pytesseract` + `pdf2image` - OCR для сканов

**Выход:**
```python
{
    'doc_id': 'abc123...',
    'pages': [
        {'page_num': 1, 'text': '...'},
        {'page_num': 2, 'text': '...'}
    ],
    'metadata': {
        'title': 'Law No. 5 of 2018',
        'date': '2018',
        'law_number': 'Law No. 5 of 2018',
        'case_number': None,
        'jurisdiction': 'DIFC'
    }
}
```

### 2. StructuralChunker (`chunker.py`)

**Назначение:** Разбивка документов на осмысленные фрагменты

**Стратегия:**
1. Поиск структурных элементов (Article, Section, пункты)
2. Разбивка по границам структуры
3. Если секция > chunk_size, дополнительная разбивка с overlap
4. Сохранение иерархии для контекста

**Параметры:**
- `chunk_size`: 512 токенов (по умолчанию)
- `overlap`: 50 токенов

**Выход:**
```python
{
    'text': 'Article 5...',
    'metadata': {
        'doc_id': 'abc123...',
        'page': 15,
        'hierarchy': ['Article 5', 'Paragraph 3'],
        'title': 'Law No. 5 of 2018',
        ...
    }
}
```

### 3. HybridIndexer (`indexer.py`)

**Назначение:** Создание векторного и BM25 индексов

**Векторный индекс:**
- Модель: `sentence-transformers/all-MiniLM-L6-v2`
- Размерность: 384
- Метрика: Inner Product (косинусное сходство)
- Библиотека: FAISS

**BM25 индекс:**
- Алгоритм: BM25 Okapi
- Токенизация: lowercase + split
- Библиотека: rank-bm25

**Гибридный поиск:**
```python
combined_score = α * vector_score + (1-α) * bm25_score
```
где α = 0.5 (по умолчанию)

### 4. Reranker (`reranker.py`)

**Назначение:** Точная переоценка релевантности

**Модель:** `cross-encoder/ms-marco-MiniLM-L-6-v2`

**Принцип работы:**
- Попарная оценка (query, document)
- Более точная, но медленная оценка
- Применяется только к top-K кандидатам

### 5. QueryRewriter (`query_rewriter.py`)

**Назначение:** Генерация альтернативных формулировок

**Методы:**
1. Расширение синонимами (юридические термины)
2. Упрощение до ключевых слов
3. Преобразование вопроса в утверждение

**Пример:**
```
Вход: "Какая ставка НДС на медицинские товары?"
Выход:
  1. "Какая ставка НДС на медицинские товары налог на добавленную стоимость"
  2. "ставка НДС медицинские товары"
  3. "ставка НДС медицинские товары"
```

### 6. AnswerGenerator (`generator.py`, `llm_integration.py`)

**Назначение:** Генерация ответа на основе контекста

**Режимы работы:**
1. **С LLM** (OpenAI API) - точные, структурированные ответы
2. **Без LLM** (эвристики) - базовая экстракция

**Типы ответов:**
- `boolean` - true/false
- `number` - числовое значение
- `date` - дата (YYYY-MM-DD)
- `name` - одно имя
- `names` - список имен
- `free_text` - текст (до 280 символов)

## Оптимизации

### Производительность

1. **Батчинг эмбеддингов** - обработка по 32 чанка
2. **FAISS IndexFlatIP** - быстрый поиск по внутреннему произведению
3. **L2 нормализация** - косинусное сходство через IP
4. **Дедупликация** - удаление дубликатов по хешу текста

### Качество

1. **Гибридный поиск** - комбинация семантического и лексического
2. **Реранкинг** - точная оценка top-K
3. **Threshold validation** - фильтрация нерелевантных результатов
4. **Query rewriting** - множественные формулировки

### Масштабируемость

1. **Индексы на диске** - не требуют пересоздания
2. **Инкрементальное обновление** - добавление новых документов
3. **Батчинг запросов** - обработка нескольких вопросов
4. **Кэширование** - эмбеддинги частых запросов

## Метрики и телеметрия

### Собираемые метрики

1. **TTFT (Time To First Token)** - время до получения контекста
2. **Total Time** - общее время обработки
3. **Token Usage** - количество токенов (prompt + completion)
4. **Retrieved Chunks** - количество найденных чанков
5. **Max Relevance Score** - максимальная оценка релевантности

### Формат телеметрии

```json
{
  "ttft_ms": 850,
  "total_time_ms": 1200,
  "token_usage": {
    "prompt": 850,
    "completion": 15
  },
  "retrieved_chunk_pages": [
    {"doc_id": "abc123...", "page": 5},
    {"doc_id": "def456...", "page": 12}
  ]
}
```

## Конфигурация

### Основные параметры

| Параметр | Значение | Описание |
|----------|----------|----------|
| CHUNK_SIZE | 512 | Размер чанка в токенах |
| CHUNK_OVERLAP | 50 | Перекрытие между чанками |
| TOP_K_RETRIEVAL | 40 | Кандидатов после поиска |
| TOP_K_RERANK | 5 | Финальных результатов |
| RELEVANCE_THRESHOLD | 0.3 | Порог релевантности |

### Модели

| Компонент | Модель | Размер |
|-----------|--------|--------|
| Embeddings | all-MiniLM-L6-v2 | 80MB |
| Reranker | ms-marco-MiniLM-L-6-v2 | 90MB |
| LLM (опц.) | gpt-3.5-turbo | API |

## Требования к ресурсам

### Индексация

- **RAM:** 4-8 GB
- **Диск:** 500 MB (индексы)
- **Время:** 10-30 минут (87 документов)

### Инференс

- **RAM:** 2-4 GB
- **Latency:** 500-2000 ms на запрос
- **Throughput:** 10-50 запросов/сек

## Расширения

### Возможные улучшения

1. **GPU ускорение** - FAISS GPU, CUDA
