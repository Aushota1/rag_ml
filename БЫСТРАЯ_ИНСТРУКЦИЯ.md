# Быстрая инструкция по использованию

## 🚀 Два режима индексации

Система поддерживает **два режима** создания индексов:

### 1️⃣ Стандартный режим (БЕЗ tokenier)

**Быстро и просто!**

```bash
# Просто запустите
python build_index.py

# Или явно укажите режим
python build_index.py --mode standard
```

✅ Не требует обучения моделей  
✅ Быстрая индексация  
✅ Подходит для прототипирования  

---

### 2️⃣ Режим с tokenier (С классификацией)

**Максимальная точность!**

#### Шаг 1: Обучить модели

```bash
# 1. Обучить BPE токенизатор (10-30 минут)
python train_bpe_tokenizer.py

# 2. Обучить embedding layer (30-60 минут)
python train_embedding_layer.py

# 3. Обучить классификаторы (опционально, 10-20 минут)
python train_tokenier_models.py
```

#### Шаг 2: Проверить готовность

```bash
python check_tokenier_setup.py
```

#### Шаг 3: Создать индекс

```bash
python build_index.py --mode tokenier
```

✅ Классификация типов документов  
✅ Семантическая сегментация  
✅ Максимальная точность  

---

## 📋 Проверка файлов

После обучения должны быть созданы:

```
models/tokenier/
├── tokenizer.pkl          # BPE токенизатор
├── checkpoint.pkl         # Чекпоинт обучения
└── embedding_model.pth    # Embedding модель
```

Проверить наличие:

```bash
ls -lh models/tokenier/
```

## 🔧 Настройка через .env

```bash
# Включить tokenier режим
USE_TOKENIER=true

# Включить классификаторы
USE_DOCUMENT_CLASSIFIER=true
USE_QUESTION_CLASSIFIER=true
USE_RELEVANCE_CLASSIFIER=true
```

## ❓ Что выбрать?

### Используйте СТАНДАРТНЫЙ режим если:
- 🚀 Нужна быстрая индексация
- 🎯 Прототипирование
- ⏰ Нет времени на обучение

### Используйте TOKENIER режим если:
- 🎯 Нужна максимальная точность
- 📚 Работа с юридическими документами
- 🏆 Production-ready решение

---

## 🆘 Проблемы?

### Ошибка: "Tokenizer not found"

```bash
# Решение: обучите токенизатор
python train_bpe_tokenizer.py
```

### Ошибка: "Embedding model not found"

```bash
# Решение: обучите embedding layer
python train_embedding_layer.py
```

### Медленная индексация

```bash
# Используйте стандартный режим
python build_index.py --mode standard
```
