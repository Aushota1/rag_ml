## 🚀 Быстрый старт

### Вариант 1: Standard Mode (рекомендуется для начала)

```bash
# Построение индекса без tokenier
python build_index.py --mode standard
```

### Вариант 2: Tokenier Mode (для production)

```bash
# 1. Проверьте файлы
ls models/tokenier/
# Должны быть: tokenizer.pkl, embedding.pth

# 2. Обучите классификаторы
python train_tokenier_models.py --model all

# 3. Постройте индекс с tokenier
python build_index.py --mode tokenier
```

## ✨ Что работает

### Standard Mode ✅
- Парсинг PDF документов
- Структурная сегментация
- Построение индекса
- Поиск и генерация ответов

### Tokenier Mode ✅
- Все из Standard Mode +
- Классификация типов документов
- Семантическая сегментация
- Улучшенное качество поиска

## 🎯 Рекомендации

### Для первого запуска:
→ Используйте **Standard Mode**

### Для production:
→ Обучите модели и используйте **Tokenier Mode**

### Для экспериментов:
→ Попробуйте оба режима и сравните

## 📞 Troubleshooting

### "Tokenizer not found"
→ См. `SETUP_TOKENIER_FILES.md`

### "Document classifier not trained"
→ Запустите `python train_tokenier_models.py --model document`

### Медленное построение индекса
→ Используйте `--mode standard` для быстрого тестирования

## 🎉 Готово!

Система готова к использованию в обоих режимах. Начните с Standard Mode для быстрого тестирования, затем переходите на Tokenier Mode для максимального качества.

**Следующий шаг:** `python build_index.py --mode standard`
