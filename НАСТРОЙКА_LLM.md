# Настройка LLM для генерации ответов

## 🎯 Текущая ситуация

Вы видите предупреждение:
```
Warning: Failed to initialize OpenAI client: The api_key client option must be set
```

Это означает, что система готова использовать мощную LLM модель, но не может найти API ключ.

## ✅ Решение (2 минуты)

### Шаг 1: Откройте файл `.env`

Файл находится здесь: `C:\Users\Aushota\Desktop\rag_ml\.env`

### Шаг 2: Найдите строку с API ключом

```env
OPENAI_API_KEY=sk-your-actual-api-key-here
```

### Шаг 3: Замените на ваш реальный ключ

```env
OPENAI_API_KEY=sk-proj-ваш-настоящий-ключ-здесь
```

### Шаг 4: Сохраните файл

### Шаг 5: Запустите снова

```bash
python hack/generate_submission.py
```

## 🔑 Где взять API ключ?

1. Откройте: https://platform.openai.com/api-keys
2. Войдите в аккаунт (или создайте новый)
3. Нажмите "Create new secret key"
4. Скопируйте ключ (показывается только один раз!)
5. Вставьте в `.env` файл

**Подробная инструкция:** см. файл `GET_API_KEY.md`

## 💰 Стоимость

Для обработки 100 вопросов:
- С моделью `gpt-4o-mini`: ~$0.10-0.30 USD
- Очень дешево и качественно!

## 🆓 Альтернатива: Без OpenAI

Если не хотите использовать OpenAI, измените в `.env`:

```env
USE_LLM=false
```

Система будет использовать эвристики (бесплатно, но менее точно).

## 🧪 Проверка настроек

После добавления ключа проверьте:

```bash
python test_llm.py
```

Должно показать:
```
✓ LLM mode enabled
  Model: gpt-4o-mini
OPENAI_API_KEY: Set
```
